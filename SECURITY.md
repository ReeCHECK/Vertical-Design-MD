# Security

Do not report vulnerabilities through public issues. Use [GitHub private vulnerability reporting](https://github.com/ReeCHECK/Vertical-Design-MD/security/advisories/new) when available, or contact the maintainers privately through their GitHub profile.

The reference packs contain design guidance, not executable production controls. Never rely on a pack alone to enforce authorization, privacy, transaction safety, or regulatory requirements; enforce those rules on trusted backend systems.

Supported versions are the latest release and the current `main` branch. Maintainers will acknowledge a valid report as soon as practical and coordinate disclosure after a fix or documented mitigation exists.
